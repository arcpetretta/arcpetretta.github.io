    • Ruby supports 5 differnet variable scopes. They are arranged in a hierarchy of sorts. If you arrange them from least powerful to most powerful they would go in the following order: local, instance, class, global and constant variables.

    Local Variables:

    Local variables are normal variables... An example would be x = 10. They always begin with a lowercase letter and does not have a nil value before initialization. The best time to use these would be when you want to set the value of variable only one time in a specific spot. If you'd like to give the variable larger scope it would be better to use a different kind of variable.

    Instance Variables:

    Instance variables begin with an @ symbol in front of the variable's name. These are usually used with classes. The most appropriate time to use this kind of variable would be when you want each instance of the class to have a different or seperate value. Ex: @nba_team.name = Knicks.

    Class Variables:

    Class variables are set with @@ symbols in front of the variables name. They work the same as instance variables accept they have the same value over all instances of the object. Ex: @@nba_team.name = Knicks.

    Global variables:

    Global variables use a $ symbol in front of the variable's name. Global variables are accessible everywhere! example: $GLOBAL_VARIABLE

    Constant variables:

    Constants always start with a capital letter. However, they are typically written in ALL CAPS. Constants counter-intuitively can have their values changed. Ruby will usually warn you before you change it.
